Class is in session - 08.04.2014

When should we use classes?
Well, before answering that question, let's take a step back. Ruby is an Object Oriented Language. That means that Ruby's language uses objects and their associated procedures within the programming context to create software programs. Object-oriented programming (OOP...yeah you know me?) refers to programming style based on objects, insted of just functions and procedures. The objects are organized into classes, which allow individual objects to be grouped together. An "object" in OOP refers to a specific type, or "instance", of a class. You can think of a class like a single concept (such as Thundercats), and an object is a single thing based on a class (such as "Lion-O" or "Cheetara"). If you want to create another concept or object called Thundercats, you can use a class. While creating the class, which I will show you later, you can add attributes to it (such as name, age, gender, weapon, etc.).

Why should we use classes?
When you are coding, you don't want 100's of different methods and variables that overlap and do the same things or else you'll just go cray cray. That's where classes come in to keep you organized. You'll just need to write one method to update your data one place at a time. It will help you save time and make life easier. If you've written a method and want to run it on a bunch of different objects; instead of copy the code from a textfile and pasting, just create a class with that method in it and call it on your object. BAM!

Say you want to make an army of 10,000 Stormtroopers. Since your run 'o the mill Stormtrooper looks exactly the same, how about using a class? I mean, you can store the value of each trooper in a varible, like a Stormtrooper's dexterity, knowledge, mechanical, perception, strength, and technical but that's soooooo repetitive and time consuming when you have to make 10k of them.

Well to make this easier on ourselves, we need to start with a template, in this case, Jango Fett. We will use this template (or class) to create more Stormtroopers (since they're all exactly the same) with all of the necessary information (or variables) then just clone it and you're ready for war! Clone Wars. Ha!

How should we use classes?
Creating a class has a particular format; kinda like computers have OCD or something. So let's get back to the Stormtrooper example. Like I mentioned before, we will create a class. The format looks like this:

class Stormtrooper <== the name of the class has to be capitalized
def initialize(dexterity, knowledge, mechanical, perception, strength, technical) <== We are initializing the object with values you will provide for each characteristic
@dexterity = dexterity <== These add variables for the object
@knowledge = knowledge
@mechanical = mechnical
@perception = perception
@strength = strength
@technical = technical
end <== Here we are ending the definition for initialize

def show_stats <== I've added this method to print out the stats
puts "Dexterity is " + @dexterity.to_s
puts "Knowledge is " + @knowledge.to_s
puts "Mechanical is " + @mechanical.to_s
puts "Perception is " + @perception.to_s
puts "Strength is " + @strength.to_s
puts "Technical is " + @technical.to_s
end
end <== This final end will close the definition of the class

regular_stormtrooper=Stormtrooper.new(5,5,5,5,5,5) <== We are creating a new regular stormtrooper with the stats
p regular_stormtrooper.show_stats <== We are calling (running) show_stats to print the stats out

You'll see:
Dexterity is 5
Knowledge is 5
Mechanical is 5
Perception is 5
Strength is 5
Technical is 5

When you're ready to create another regular Stormtrooper, you'll just need another unique name. Notice that the stats are the same:

regular_stormtrooper2=Stormtrooper.new(5,5,5,5,5,5)
p regular_stormtrooper2.show_stats

You'll see:
Dexterity is 5
Knowledge is 5
Mechanical is 5
Perception is 5
Strength is 5
Technical is 5

If you feel like a saucy master cloner, like Lama Su, you can adjust the stats and give them a new name to build a new army: imperial_trooper=Stormtrooper.new(2,2,2,2,2,2)
p imperial_trooper.show_stats

You'll see:
Dexterity is 2
Knowledge is 2
Mechanical is 2
Perception is 2
Strength is 2
Technical is 2

tie_pilot=Stormtrooper.new(2,2,3,3,2,2)
p tie_pilot.show_stats

You'll see:
Dexterity is 2
Knowledge is 2
Mechanical is 3
Perception is 3
Strength is 2
Technical is 2

VIOLA! That's it. You've created a simple class and created new objects.
