What's the deal with Margin, Border, and Padding?
Well, the margin clears out the area around an "element" (an HTML element is everything from the start tag to the end tag) outside the border. You won't be able to see the margin because it doesn't have a background color and it's transparent (think of it as a force field). You can define margins like this: p {margin-top: px; margin-bottom: px; margin-right: px; margin-left: px;}.

A border, on the other hand, is a style property that allows you to set what kind of border to display. In order to see the border, first you must decide how you want to display the border. Would you like your border to be dotted, dashed, solid, double (lined), groove, ridge, inset, or outset. You'd also have to define how wide the border is by using pixels. Finally, you have to choose a color using a hexadecimal or hex (#), or RGB notation. So you can define a border like this: p {border: ; border-width: ; border-style: ; border-color: ;}.

Finally, padding clears the area around the content of an element inside the border. The padding can be affected by the background color of the element. Yopu can also define the top, right, bottom, and left property by using pixels. This is how you can define padding: p {pading-top: px; padding-bottom: px; padding-right: px; padding-left: px;}

What are the best practices associated with using classes versus ids?
When you are trying to format a particular element, say body, you can only use one ID within it. If you want to use only one style/format then you're good because IDs are unique. This means that each element can only have one ID AND each page can have only one element with that ID.

If you want to make the body element saucy and apply more than one style, then you're gonna need to use classes because they're not unique. That means that you can use the same class on multiple elements AND you can use multiple classes on the SAME element. If you have a class named "awesome", you can use it over and over again on your HTML page.

One cool thing is that an element can have both an ID and a class. For example, li id="awesome" class="sauce"

What are the differences between relative, absolute, and fixed positioning?
The relative positioning property value is based on the element's position relative to it's normal position. Therefore, if you select "right: 20px", the setting will add 20 pixels to the element's RIGHT position.

The absolute positioning property value will allow you to place any element where you want it. You can use positioning options such as top, left, bottom, and right to set the location. The important thing to remember is that these positioning options are based relative to the next parent element's relative position on the page. If there are no parent element's, the default setting will place the element at th etop of the page. Another important thing to remember is that absolute positioning will remove the element from the flow of the other elements of the page and will not be affected by other elements.

A fixed type of positioning is rarely used because the position of the element is relative to the viewport, or the browser window itself. It would work if the viewport stays the same, but people have different ways they can view your work, thus their experience will vary. You don't want to limit your audience to use one sized monitor to see your site properly, right?
